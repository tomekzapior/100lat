# Styl tekstów PL

Dotyczy UI, dokumentacji, README i komunikatów.

Zakazane słowa: kluczowy, istotny, fundamentalny, wszechstronny, złożoność, dynamiczny, przełomowy, rewolucyjny, zachwyca.

Zakazane czasowniki: zgłębiać, uwypuklać, podkreślać, ukazywać, prezentować.

Zakazane konstrukcje: „to nie tylko X, ale także Y”, „nie X, lecz Y” oraz imiesłowy dopisane na końcu zdania.

Zasady:

- Bez em dash. Używaj przecinka, dwukropka, nawiasu albo zwykłego dywizu.
- Bez emoji w formalnym UI i dokumentacji.
- Nagłówki mają wielką tylko pierwszą literę.
- Pogrubienia stosuj oszczędnie.
- Używaj jednego terminu konsekwentnie zamiast kilku synonimów.
- Komunikaty opisują, co się stało i co można zrobić dalej.
